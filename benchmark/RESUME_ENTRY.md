**高性能HTTP服务器**

项目仓库：https://gitee.com

项目简介：基于 Reactor 多线程模型实现的高性能 HTTP 服务器，支持 HTTP/1.1 协议、Keep-Alive 连接复用、正则路由分发及静态文件服务。采用 One Loop Per Thread 架构将网络 IO 与业务处理解耦，epoll 边缘触发驱动事件分发，时间轮管理连接生命周期，可用于快速搭建高并发 Web 服务。

核心工作：

- 采用 "one thread one loop" 思想设计主从 Reactor 模型，主线程负责 accept 新连接并按负载均衡分发给从线程池（4个 IO 线程），每个线程独立运行 epoll 事件循环处理 IO 事件，实现多核并行处理。

- 基于 epoll ET（边缘触发）模式实现高效事件分发，配合非阻塞 IO + Buffer 缓冲区设计，减少系统调用次数与上下文切换开销；通过 eventfd 实现跨线程任务投递与唤醒机制。

- 实现时间轮（TimeWheel）定时器结构管理空闲连接回收，支持可配置超时自动断开，避免文件描述符泄漏；采用智能指针管理 Connection 生命周期，保证连接安全释放。

- 实现状态机驱动的 HTTP 请求解析器，支持分段解析与错误容错；封装 HttpRequest/HttpResponse 对象，支持 URL 解码、查询参数提取、正则路由匹配（std::regex）、静态文件 MIME 类型识别等完整 HTTP 功能。

项目成果：

- 在 4 核 4GB 云主机环境下，使用 Webbench 压测工具进行多维度测试（低/中/高/极限四档并发梯度 + 协议对比 + 120 秒长时间稳定性），简单路由响应峰值 QPS 达 **9,900+**，稳定 QPS 达 **9,274**（120 秒持续高压下 0 失败）。

- 并发连接数在 **2,500** 以内零失败稳定运行，极限阈值约 **3,000** 连接；HTTP/1.1 Keep-Alive 相比 HTTP/1.0 短连接性能提升约 **11.6%**，静态文件服务峰值带宽达 **18.9 MB/s**。
