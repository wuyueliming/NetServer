# 高性能HTTP服务器性能测试报告

## 一、测试环境

| 项目 | 配置 |
|------|------|
| CPU | Intel Xeon Platinum 8255C @ 2.50GHz, 4核 |
| 内存 | 4GB DDR4 |
| 操作系统 | Ubuntu 22.04, Linux 5.15.0 |
| 编译器 | g++ 11.4.0, -O2优化 |
| 测试工具 | Webbench 1.5 |
| 服务器架构 | Reactor多线程模型, 4 IO线程, epoll边缘触发 |

## 二、服务器架构概述

基于muduo风格的高并发HTTP服务器，核心特性：
- **One Loop Per Thread**：主Reactor负责接受连接，4个从Reactor线程处理IO
- **epoll ET模式**：边缘触发，减少系统调用次数
- **时间轮踢掉空闲连接**：60秒超时自动断开
- **正则路由分发**：支持GET/POST/PUT/DELETE及静态文件服务
- **连接保活**：HTTP/1.1 Keep-Alive支持

## 六、性能分析

1. **日志锁是关键瓶颈**：关闭日志后QPS从9,305提升至9,908（+6.9%），证明mutex竞争是主要瓶颈之一
2. **吞吐量 plateau 特征**：并发数从50到2000，QPS稳定在9,200~9,900区间，4核CPU是主要瓶颈
3. **小响应体CPU密集**：/ping和/hello等小响应场景，QPS受限于HTTP协议解析和epoll事件处理开销
4. **大响应体带宽密集**：静态文件和1KB响应场景，带宽达到18MB/s，受限于loopback接口
5. **连接管理健壮性**：c≤2500时0失败，体现了时间轮+epoll ET的连接管理可靠性
6. **高并发崩溃点**：c≥3000时服务器不稳定，主要原因是进程文件描述符和TCP连接队列达到系统限制

## 七、简历描述建议

> 基于Reactor模式实现高性能HTTP服务器，采用One Loop Per Thread架构、epoll边缘触发、时间轮连接管理等技术。经Webbench压力测试，4核服务器下峰值QPS达9,900+，120秒持续高压（1000并发）测试0失败，安全并发连接数2,500+。支持HTTP/1.1 Keep-Alive连接复用，较HTTP/1.0性能提升11.6%；支持正则路由分发、静态文件服务、超时连接回收等功能。
